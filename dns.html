Definisi DNS (Domain Name System)

DNS (Domain Name System) adalah sistem terdistribusi yang berfungsi sebagai "buku telepon internet". Peran utamanya adalah menerjemahkan Nama Domain yang mudah diingat manusia (seperti google.com) menjadi alamat IP (Internet Protocol) yang dipahami komputer (seperti 172.217.14.228).

Komputer di internet saling menemukan menggunakan angka (alamat IP), bukan nama. DNS adalah layanan penerjemah yang menjembatani kesenjangan tersebut.

Bagaimana Cara Kerjanya? (Alur DNS Lookup)

Gambar di atas mengilustrasikan alur pencarian DNS (DNS Lookup) yang detail. Saat kamu mengetik example.com di browser:

  1. DNS Query (Langkah 1): Browser-mu (User) bertanya ke "Recursive DNS Resolver" (biasanya disediakan oleh ISP-mu), "Apa IP untuk example.com?"
  2. Recursive Query (Langkah 2-7): Resolver-mu (jika tidak punya di cache) memulai pencarian berantai:
    • Langkah 2 & 3: Resolver bertanya ke Root DNS Server ("Hei, di mana server .com?"). Root Server menjawab dengan alamat TLD Server.
    • Langkah 4 & 5: Resolver bertanya ke .com TLD Name Server ("Hei, di mana server untuk example.com?"). TLD Server menjawab dengan alamat Authoritative Server.
    • Langkah 6 & 7: Resolver bertanya ke Authoritative DNS Resolver (server DNS resmi milik example.com).
  3. Jawaban (Langkah 8): Authoritative Server memberikan jawaban pasti: "IP-nya adalah 192.0.0.16" (contoh di gambar).
  4. Selesai: Resolver memberikan IP itu ke browser-mu. Sekarang browser-mu tahu ke mana harus mengirim permintaan HTTP.
